# 🧭 VaultLite V2 项目方案（开发者路线）

> **Local-first 密码保险箱 · GitHub 自动备份 · CLI 优先**
> 定位：开发者密码管理器——纯本地、GitHub 托管、零配置

---

## 1. 项目本质

VaultLite 是一个：

> **面向开发者的极简密码管理器：纯前端加密 + GitHub 私有仓库自动备份 + CLI 工具链**

### 核心差异化

| 维度 | Bitwarden / 1Password | VaultLite |
|------|----------------------|-----------|
| 存储 | 自有云 | GitHub 私有仓库 |
| 使用方式 | GUI 为主 | CLI 优先，Web 为辅 |
| 版本历史 | 无 | Git commit history |
| 开源 | Bitwarden 是 | VaultLite 也是 |
| 用户 | 所有人 | 开发者 |
| 安装 | 需要注册 | 一杯咖啡 + GitHub 登录 |

---

## 2. 设计原则

### P1: 本地优先
- 所有数据先在浏览器/本地加密
- IndexedDB 是运行态存储
- 离线完全可用

### P2: 零信任后端
- GitHub 只存密文 `vault.enc`
- 不参与加解密
- 不知道主密码

### P3: CLI 优先，Web 为辅
- 核心操作 CLI 可完成
- Web 作为可视化辅助
- 管道友好（`vault get xxx | pbcopy`）

### P4: Git-native 备份
- 每个备份是一个 git commit
- 天然版本历史
- 可 diff、可回滚

## 4. 功能范围

---

### 🔐 ① 密码保险箱（Vault）

#### 记录字段

```json
{
  "id": "uuid",
  "title": "GitHub",
  "username": "user@example.com",
  "password": "encrypted",
  "url": "https://github.com",
  "note": "personal account",
  "tags": ["dev", "social"],
  "createdAt": "ISO8601",
  "updatedAt": "ISO8601"
}
```

#### 功能
- 新增 / 编辑 / 删除 / 查看
- 一键复制（CLI: `vault get github -c password`）
- 搜索（标题 / URL / 标签 / 用户名）
- 标签分类

---

### ⌨️ ② CLI 工具（核心差异化）

```
Usage:
  vault init                    # 初始化 vault + GitHub repo
  vault list                    # 列出所有条目
  vault get <query> [-c field]  # 获取条目，-c 复制到剪贴板
  vault add                     # 交互式新增
  vault edit <id>               # 编辑条目
  vault rm <id>                 # 删除条目
  vault sync                    # 手动触发备份
  vault status                  # 显示同步状态
  vault config                  # 查看/修改配置
  vault export                  # 导出 vault.enc
  vault import <file>           # 导入 vault.enc

管道示例:
  vault get github -c password          # 复制到剪贴板
  vault get github -f json | jq .       # JSON 输出
  vault list --format csv > backup.csv  # 导出 CSV
```

CLI 使用 **Go** 编写（单二进制分发，跨平台），通过 Web 页面暴露的 HTTP API 或直接操作 IndexedDB 的数据映射。

---

### 🔑 ③ 密码生成器

- 长度调节
- 大写 / 小写 / 数字 / 符号
- 排除易混字符（il1o0O）
- 可读密码（passphrase 模式）
- CLI: `vault generate --length 24 --no-symbols`

---

### 📊 ④ 密码强度检测

- 强度评分（0–100）
- 等级：Weak / Medium / Strong / Very Strong
- 字符覆盖分析
- 熵估算
- 泄露检测（本地 hash 查 Have I Been Pwned API，k-anonymity）

---

### 💾 ⑤ 自动备份系统

#### 触发条件
- 新增 / 修改 / 删除密码
- 手动锁定
- 导入 vault
- 页面切后台（可选）
- CLI 显式 `vault sync`

#### 防抖策略
```
修改发生 → 等待 30 秒 → 自动备份
```

#### 去重机制
```
currentHash === lastBackupHash → 不上传
```

#### GitHub 仓库结构
```
vaultlite-backup (private repo)
├── vault.enc          # 当前加密数据
└── vault.changelog    # 可读变更摘要（明文，仅元数据）
```

每次备份 = git commit：
```
commit a1b2c3d4
Author: VaultLite <vault@local>
Date:   Tue Jun 30 2026

Update vault.enc — 3 entries, 1 change
```

---

### 💽 ⑥ 导入 / 导出

- 导出：`vault.enc` / CSV / JSON 明文（有警告）
- 导入：拖拽 `vault.enc` 或 `vault import`
- 支持 Bitwarden CSV 导入（迁移友好）

## 5. 加密系统

```
Master Password
    ↓
PBKDF2 (SHA-256, 600k iterations — OWASP 2026 推荐)
    ↓
AES-256-GCM
    ↓
vault.enc 存储格式: nonce || ciphertext || tag
```

- 主密码永不上传
- 密钥只存在内存
- 关闭页面/进程即清空
- 未来可选 Argon2id（WASM 实现）

## 8. 用户完整流程

### 第一次使用
```
打开网页 / 下载 CLI
→ 创建主密码
→ GitHub OAuth 登录
→ 自动创建 vaultlite-backup 私有仓库
→ 初始化完成
```

### 日常使用（CLI）
```
vault list
→ vault get github -c password  (直接到剪贴板)
→ vault add                     (交互式新增)
→ 自动 30 秒后备份到 GitHub
```

### 日常使用（Web）
```
打开网页 → 解锁 → 管理密码 → 自动备份
```

### 换设备
```
安装 CLI / 打开网页
→ GitHub OAuth 登录
→ 检测 vault.enc
→ 输入主密码 → 解密 → 恢复
```

### 团队/家庭分享（V2 考虑）
```
vault share <id> --with <github-user>
→ 端到端加密分享
→ 接收方用自己的主密码解密
```

## 11. V1 不做（保持锐利）

| 不做 | 原因 |
|------|------|
| ❌ 手机 App | 开发者主要在桌面 |
| ❌ 自动填充 | 浏览器插件需要额外维护 |
| ❌ 云同步 | 违背零信任原则 |
| ❌ 多用户/团队 | V2 的事 |
| ❌ OTP/TOTP | 不是开发者核心需求 |
| ❌ 自有付费计划 | 先做名气，变现不着急 |
